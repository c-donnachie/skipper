# Aplicar perfil de stack

Aplicar el stack "$ARGUMENTS" al proyecto actual.

## Validación previa

Si "$ARGUMENTS" está vacío:
1. Sugiere correr `/skipper:scan` primero.
2. Pide al usuario el id del stack.
3. Termina sin escribir nada.

Verifica que existe el directorio `${CLAUDE_PLUGIN_ROOT}/stacks/$ARGUMENTS/`. Si no existe, reporta los stacks disponibles (lista `${CLAUDE_PLUGIN_ROOT}/stacks/`) y termina.

## Pasos

### 1. Lee el perfil

Lee `${CLAUDE_PLUGIN_ROOT}/stacks/$ARGUMENTS/profile.json`. Te da metadata del stack (nombre, descripción, libs principales).

### 2. Lee templates

- `${CLAUDE_PLUGIN_ROOT}/stacks/$ARGUMENTS/claude.md.tmpl` — fragmento opinado para CLAUDE.md
- `${CLAUDE_PLUGIN_ROOT}/stacks/$ARGUMENTS/architecture.md.tmpl` — doc de arquitectura inicial

### 3. Sustituye variables en ambos templates

| Variable | Valor |
|---|---|
| `{{DATE}}` | Resultado de `date +%Y-%m-%d` |
| `{{AUTHOR}}` | Resultado de `git config user.name` (fallback: "TODO") |
| `{{PROJECT_NAME}}` | Nombre de la carpeta actual (`basename $(pwd)`) |
| `{{STACK_ID}}` | $ARGUMENTS |

### 4. Aplica a CLAUDE.md (idempotente)

Marcadores: `<!-- skipper:stack -->` y `<!-- /skipper:stack -->`.

**Si CLAUDE.md no existe**: créalo con sólo el contenido del template entre marcadores.

**Si CLAUDE.md existe Y contiene marcadores skipper:stack**: reemplaza la sección entre marcadores con el contenido nuevo. NO toques nada fuera de los marcadores.

**Si CLAUDE.md existe pero NO tiene marcadores**: agrega el bloque al FINAL del archivo (con una línea en blanco antes), envuelto en marcadores. NO modifiques contenido existente.

Formato del bloque a insertar/reemplazar:
```
<!-- skipper:stack -->
<!-- generated by skipper v0.2.0 — stack: {{STACK_ID}} — {{DATE}} -->

[contenido sustituido del claude.md.tmpl]

<!-- /skipper:stack -->
```

### 5. Crea/actualiza docs/architecture/stack.md

`mkdir -p docs/architecture`. Si `docs/architecture/stack.md` no existe, escríbelo desde el template sustituido. Si existe, **NO sobreescribas** — sólo reporta y sigue.

### 6. Actualiza docs/index.md (si existe)

Si `docs/index.md` existe y no menciona `architecture/stack.md`, agrega un link en la sección Architecture.

### 7. Reporta

Tabla:

| Acción | Path | Resultado |
|---|---|---|
| Generado | CLAUDE.md (sección skipper:stack) | nueva / actualizada |
| Creado | docs/architecture/stack.md | sí / saltado (existía) |
| Linkado | docs/index.md | sí / saltado |

Después invita al usuario a leer CLAUDE.md y validar las convenciones declaradas.

## Reglas duras

- **NUNCA** sobreescribas contenido fuera de los marcadores `<!-- skipper:stack -->`.
- **NUNCA** sobreescribas `docs/architecture/stack.md` si ya existe — el usuario podría haberlo customizado.
- Si "$ARGUMENTS" no es un stack válido, NO escribas nada y reporta.
- Idioma del template: español (todos los stacks v0.2 usan español chileno informal).
