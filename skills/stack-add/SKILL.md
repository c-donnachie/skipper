# Skipper :: stack-add

Agregar el layer "$ARGUMENTS" al CLAUDE.md del proyecto.

## Validación previa

Si "$ARGUMENTS" está vacío:
1. Lista los layers disponibles: `ls "${CLAUDE_PLUGIN_ROOT}/stacks/_layers/"`.
2. Pide al usuario que indique cuál.
3. Termina sin escribir.

Verifica que existe `${CLAUDE_PLUGIN_ROOT}/stacks/_layers/$ARGUMENTS/layer.json`. Si no, reporta los layers disponibles y termina.

Verifica que existe `CLAUDE.md` en el proyecto. Si no, sugiere `/skipper:stack-apply` primero y termina.

## Pasos

### 1. Lee el layer

- `${CLAUDE_PLUGIN_ROOT}/stacks/_layers/$ARGUMENTS/layer.json` (metadata, `compatible_with`, `requires`).
- `${CLAUDE_PLUGIN_ROOT}/stacks/_layers/$ARGUMENTS/fragment.md` (contenido a insertar).

### 2. Valida compatibilidad

Lee la sección skipper:stack del CLAUDE.md actual para detectar el stack. Si el stack del proyecto NO está en `compatible_with` del layer, advierte al usuario y pide confirmación antes de seguir.

Si `requires` lista layers que aún no están instalados (ej. shadcn-ui requiere tailwind), advierte y pide confirmación.

### 3. Idempotencia

Marcadores: `<!-- skipper:layer:$ARGUMENTS -->` y `<!-- /skipper:layer:$ARGUMENTS -->`.

- Si CLAUDE.md ya tiene esos marcadores: reemplaza el contenido entre ellos con el fragment nuevo.
- Si NO tiene esos marcadores: agrega el bloque al final del archivo (después del último `<!-- /skipper:* -->` que encuentres), envuelto en marcadores.

Formato del bloque:
```
<!-- skipper:layer:$ARGUMENTS -->
<!-- generated by skipper v0.6.0 — layer: $ARGUMENTS — {{DATE}} -->

[contenido del fragment.md]

<!-- /skipper:layer:$ARGUMENTS -->
```

### 4. NO toques otras secciones

- No modifiques contenido fuera de marcadores `<!-- skipper:layer:$ARGUMENTS -->`.
- No toques la sección `<!-- skipper:stack -->`.
- No toques otros layers ya presentes (`<!-- skipper:layer:<otro> -->`).

### 5. Reporta

Tabla:

| Acción | Path | Resultado |
|---|---|---|
| Layer agregado | CLAUDE.md (sección skipper:layer:$ARGUMENTS) | nueva / actualizada |
| Compatibilidad con stack | $ARGUMENTS vs stack actual | OK / warning |
| Dependencias | layers `requires` | OK / faltan |

Después invita al usuario a leer CLAUDE.md con la nueva sección agregada.

## Reglas duras

- **NUNCA** sobreescribas contenido fuera de los marcadores específicos del layer.
- Sin CLAUDE.md → no escribas nada, sugiere `/skipper:stack-apply` primero.
- Si `compatible_with` no incluye el stack actual y el usuario igual lo quiere, agrégalo con un comentario `<!-- warning: layer no oficialmente compatible -->`.
